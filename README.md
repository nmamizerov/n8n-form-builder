# n8n Form Builder

Плагин Claude Code, который позволяет **нетехническим менеджерам** создавать в n8n
воркфлоу-обработчики форм, отвечая на вопросы на человеческом языке. Менеджер не пишет
JSON и не открывает редактор n8n.

Каждый создаваемый воркфлоу устроен одинаково:

```
Webhook (приём POST с формы)
   → Code (валидация полей по вашим правилам)
       → IF (данные валидны?)
           ├─ да  → ПАРАЛЛЕЛЬНО во все источники сразу: amoCRM + Telegram + Postgres
           └─ нет → алерт в Telegram (чтобы заявка не потерялась)
```

## Как это работает

Claude Code общается с n8n не напрямую, а через MCP-сервер
[`czlonkowski/n8n-mcp`](https://github.com/czlonkowski/n8n-mcp). Плагин добавляет
поверх него мастер-оркестратор: задаёт вопросы, собирает воркфлоу под ваш запрос,
показывает план и создаёт воркфлоу только после вашего апрува.

## Установка

```
/plugin marketplace add <owner>/n8n-form-builder
/plugin install n8n-form-builder
```

После установки доступна команда:

```
/create-form-handler
```

При первом запуске мастер сам проверит MCP-сервер n8n, при необходимости установит его
и попросит:

- **N8N_API_URL** — адрес вашего инстанса n8n (например `https://acme.app.n8n.cloud`)
- **N8N_API_KEY** — ключ из n8n → **Settings → n8n API → Create API key**

### Где хранятся креды

По умолчанию `.mcp.json` подставляет ключ из переменных окружения
`N8N_API_URL` / `N8N_API_KEY` — секрет не попадает в файлы и git. Мастер `n8n-setup`
поможет их задать. Если вы предпочитаете вписать ключ прямо в `.mcp.json`, раскомментируйте
строку `.mcp.json` в `.gitignore`, чтобы файл с ключом не уехал в репозиторий.

## Добавление своих примеров (повышение точности)

Точность сборки растёт за счёт **библиотеки реальных примеров** в [`examples/`](examples/).
Чтобы добавить свой эталон:

1. Экспортируйте готовый воркфлоу из n8n (**⋯ → Download**) — получится `.json`.
2. Положите файл в `examples/`.
3. Добавьте строку в [`examples/README.md`](examples/README.md): имя файла → кейс → какие ноды/назначения.

Менять код скиллов не нужно — мастер начнёт учитывать новый пример автоматически.

## Состав плагина

| Скилл | Назначение |
|-------|-----------|
| `create-form-handler` | Главный мастер (слэш-команда), ведёт менеджера по шагам |
| `n8n-setup` | Проверка/установка MCP n8n и ввод кредов |
| `n8n-build-from-examples` | Как собирать воркфлоу через MCP, опираясь на `examples/` |

## Ограничения v1

- Один шаблон — «Обработчик форм». Один общий инстанс n8n Cloud.
- Триггер — Webhook (внешняя форма). При ошибке валидации — алерт в Telegram.
- amoCRM подключается через community-ноду [`n8n-nodes-amocrm`](https://github.com/yatolstoy/n8n-node-amocrm)
  — она работает **только на self-hosted n8n** (или Enterprise Cloud). На обычном n8n Cloud
  unverified community nodes недоступны.
