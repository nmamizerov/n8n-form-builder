# Библиотека примеров воркфлоу

Скилл `n8n-build-from-examples` использует эти `.json` как образцы структуры и стиля
(few-shot), чтобы собирать воркфлоу максимально точно. Это **референсы**, а не готовые
шаблоны для слепой подстановки — реальные значения (поля, chat_id, поддомен, таблица)
всегда собираются у менеджера заново.

## Как добавить свой пример

1. Экспортируй готовый воркфлоу из n8n: открой воркфлоу → меню «⋯» → **Download**.
2. Положи полученный `.json` в эту папку.
3. Добавь строку в таблицу ниже.

Менять код скиллов не нужно — новый пример подхватится автоматически.

## Индекс примеров

| Файл | Кейс | Триггер | Ноды | Назначения |
|------|------|---------|------|-----------|
| `form-to-3-destinations.json` | Эталон: лид с формы во все источники сразу + алерт на невалид | Webhook (POST) | Webhook → Code (валидация) → IF → fan-out | amoCRM + Telegram + Postgres; ветка ошибки → Telegram-алерт |

> Плейсхолдеры в эталоне (`YOUR-SUBDOMAIN`, `YOUR_CHAT_ID`, таблица `public.leads`,
> путь `lead-acme`) заменяются на данные конкретного менеджера при сборке.
